EXPONENTIAL BACKOFF SCHEDULE:
  Attempt 1: 0s      (immediate)
  Attempt 2: 1s      (wait 1 second)
  Attempt 3: 2s      (wait 2 seconds)
  Locked:    1800s   (wait 30 minutes)

SECURITY BENEFITS:
  • Credential stuffing attacks: 3-attempt lockout prevents rapid guessing
  • Password spraying: Per-IP tracking detects distributed attacks
  • Brute force: 30-minute lockout with exponential backoff delay
  • Distributed protection: Redis-backed for multi-pod deployments

IMPLEMENTATION:

  - Already had full Sentry SDK initialization
  - Enhanced with new security event capture functions:
    • captureBruteForceAttempt(identifier, ip, attempts)
    • captureAccountLockout(identifier, ip, reason)
    • captureCsrfMismatch(sessionId, ip)
    • captureXssAttempt(payload, field, ip)
    • captureSqlInjectionAttempt(query, ip)
    • captureApiAbuse(agentId, endpoint, requestsPerSecond)
    • captureUnauthorizedAccess(agentId, resource, ip)
    • captureSuspiciousLogin(agentId, reason, data)
    • captureError(error, context)
    • addBreadcrumb(message, data, level)

  File: backend/src/middleware/sentry-middleware.ts (NEW)
  - sentryTransactionMiddleware:
    • Starts transaction per request
    • Tracks response time and status code
    • Flags slow requests (>1s)
    • Sets transaction status (ok/invalid_argument/internal_error)
  
  - sentrySecurityContextMiddleware:
    • Sets authenticated user ID
    • Tracks IP addresses
    • Captures security context
  
  - sentryAuthTrackingMiddleware:
    • Tracks auth endpoint calls
    • Creates breadcrumbs for auth flow debugging
  
  - sentryErrorContextMiddleware:
    • Captures errors with request context
    • Tags errors appropriately

  File: backend/src/server.ts
  - Import initializeSentry
  - Initialize BEFORE other middleware:
    initializeSentry(app);
  
  - Add Sentry middleware early in request pipeline:
    app.use(sentryTransactionMiddleware);
    app.use(sentrySecurityContextMiddleware);
    app.use(sentryAuthTrackingMiddleware);

MONITORING CAPABILITIES:
  • Error tracking: Unhandled exceptions captured automatically
  • Performance: Transaction tracing, P50/P95/P99 response times
  • Security events: Brute force, CSRF, XSS, unauthorized access
  • Breadcrumbs: Detailed context for debugging
  • User tracking: IP, agent ID, request details
  • Sampling: 10% in production, 100% in development
