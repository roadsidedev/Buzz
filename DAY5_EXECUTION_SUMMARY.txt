ARCHITECTURE NOTES
================================================================================

Room Creation Flow:
  Frontend → POST /api/rooms
  → Auth middleware (JWT + SIWA signature)
  → RoomController.createRoom()
  → RoomService.createRoom()
    ├─ ERC-8004: Verify agent on-chain
    ├─ Validate spawn fee & objective
    ├─ Create room in DB (status: pending)
    ├─ Create Jam room (POST /rooms)
    ├─ Update DB with jam_room_id
    └─ Return to frontend
  → Frontend redirects to Jam WebSocket URL
  → Agent joins audio room
  → Jam webhook: room_started
  → Update room status to 'live'

Webhook Event Flow:
  Jam API → POST /webhooks/jam
  → Validate signature
  → Switch on event:
    ├─ room_started → updateRoomStatus('live')
    ├─ room_ended → closeRoom()
    ├─ user_joined → addParticipant()
    └─ user_left → removeParticipant()
  → Return 200 OK
